Background

 

BrainVoyager is a powerful software system for working with neuroimaging data. Its uses include viewing, organizing, preprocessing, and analyzing fMRI data. This tutorial is designed to be a gentle introduction to some of the most common uses of BrainVoyager and their corresponding file types. To complete this tutorial, it is recommended that you use BrainVoyager 20.6 for any supported operating system. The data accompanying this tutorial includes an anatomical scan, a functional localizer scan, and an experimental protocol file. Both scans were collected from a single participant as part of a multi-subject scanning project.

 

Before getting started, recall that in most fMRI sessions, an anatomical scan is obtained in addition to any functional scans. In this tutorial, we are using scans for one participant collected at the Robarts CFMM using the 3T scanner. This scanner records and stores anatomical and functional images using the DICOM digital image format – a standard medical imaging format for both storage and transmission. Recall that an anatomical scan is recorded as a collection of 2D images or slices of the brain. This results in a separate DICOM file for each slice. Using Horos, a free MacOS application for connecting to DICOM servers, we can easily access the DICOM files for our scan. The following image is a rendering of a single DICOM file corresponding to 1 of the 176 slices of this participant’s anatomical scan. As we can see, this is in fact slice 87 (Im: 87/176), which is roughly mid-brain from left to right.

 

 

 

Other information can also be found in the above image, such as slice thickness in mm (1 mm), 2D image size in pixels (224 x 256 pixels), the participant’s identifier code (AQ18), and the name of the study (Culham Psych_9223).

 

 

Functional scans are also recorded one slice at-a-time, but unlike anatomical scans, they are stored a collection of 2D representations of an entire volume. The following is an example of a single functional DICOM file visualized using Horos.

 

 

As we can see, this image contains 52 slices of functional data, and is the first image in a sequence of 340. Together, these 52 slices provide functional data for the entire volume (in this case, the whole brain) at roughly the same point in time.

 

From DICOMs to BrainVoyager Files

 

Understanding what the raw data looks like straight from the scanner is crucial for understanding how and why BrainVoyager represents fMRI data using other formats. In this tutorial, we will consider the following file formats:

 

·       VMR – Anatomical 3D Data Set – Three-dimensional data structure for anatomical scans.

·       FMR – Functional MRI Data Set – Simple text file containing information about functional scans.

·       STC – Slice Time Course – Functional time series data organized as slices over time

·       VTC – Volume Time Course – Functional time series data organized by volumes over time.

·       PRT – Stimulation Protocol – Simple text file containing information about experimental stimulus timing.

Opening an FMR file also opens associated AMR and STC files.

 

The FMR file only contains information about the functional scan, including information from the source DICOM files. Much of this information is displayed in the FMR Properties window. To see the rest, try opening AQ18_Loc1-S1R1.fmr in a text editor like Notepad or TextEdit.

 

The AMR file (AQ18_Loc1-S1R1_firstvol_as_anat.amr) is automatically opened with the FMR file. AMR files are, according to the BrainVoyager documentation, a stack of anatomical slices that can be viewed in 2D – much like the original functional DICOM files we can view in Horos. You might notice, though, that these images are not anatomical. How can you tell? To make visualization more convenient, BrainVoyager sometimes stores functional data using file formats mostly meant for anatomical data. This AMR file is storing the first volume of functional data as if it were an anatomical scan to make it easier to visualize.

 

Finally, an STC file is also opened behind the scenes. This file is the Slice Time Course of our functional localizer, and it will allow us to do some basic quality assurance in the next step.

 

4.     Close the FMR Properties window, then select the Options menu, and Time Course Movie. Then click Preload All, wait roughly 5 seconds, and click > to visualize each the functional data over time.

 

 

This is a simple, important step for performing basic quality assurance of your functional data. What might you determine by viewing the Time Course Movie? Do you think this is good quality data? Why or why not?

 

Now that we have successfully opened and viewed both the anatomical and functional data, we can begin steps towards basic analysis. Our next step will be to align the functional data with the anatomical scan, so that we can use the participant’s brain anatomy to guide preliminary functional analysis.

 

5.     Close the FMR file tab. Look for the 3D Volume Tools window. It may already be on your screen. If it isn’t, click the blue cube to the left of the main window.

a.     Then click Full Dialog > in the 3D Volume Tools Window.

 

 

b.     Next click the Coregistration tab, and click the Select FMR… button. Select and open AQ18_Loc1-S1R1.fmr, then click Align…

 

c.     Now the VMR-FMR Coregistration window is open. By default, BrainVoyager will apply two transformation steps to align the functional and anatomical data. For future use and reproducibility, the mathematical parameters used in both transformations will be stored in two separate TRF files (see Resulting transformation files) below. Click GO to apply transformations.

 

 

After this step has completed, a collection of new TRF and VMR files will have been generated automatically. These files were used in calculations for coregistration and may be ignored in this tutorial.

 

By default, a blend of the aligned functional and anatomical data will be shown in the main window. Close or move the 3D Volume Tools Window and use the crosshairs to explore the aligned data. Was this functional-anatomical alignment successful? Why or why not?

 

In the next steps, we will transform the functional data into a Volume Time Course that will allow us to explore voxel-wise functional signal. VTC files are similar to STC files but differ in their internal organization of functional data.

 

 

6.     Close the 3D Volume Tools window. Click the Analysis menu then select Create Normalized VTC from FMR Data… Once the Create VTC window opens, use the first Browse… button to select the FMR file, then click the Auto-Fill button and BrainVoyager will auto-fill the rest of the form with the TRF files generated by the previous step. Finally, click GO.

 

 

 

It may take a few minutes before the VTC file is ready. Wait until the computation progress bar reaches 100% - it will disappear once the file is ready.

 

Once the computation is finished, we can attach the VTC to the anatomical scan and begin to explore voxel-wise functional data as time series.

a.     Select the Analysis menu and click Link Volume Time Course (VTC) File… Once the Link Volume Time Course (VTC) window appears, click the Browse button and select the VTC file created in the previous step. Click OK to close the window.
