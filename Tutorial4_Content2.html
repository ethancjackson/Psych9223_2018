General Instructions

 

Preprocessing and file conversions can take a lot of time to run – too long to do in a two-hour tutorial. This tutorial assumes that preprocessing steps have already been applied by following Instructions for Completed Steps found at the end of this page. You are encouraged to follow these steps on any data of your choice outside of class. In particular explore VMR-FMR coregistration (anatomical alignment) to get a sense of how successful or unsuccessful this process can be.

 

You are provided with a single anatomical scan in native brain space – i.e. no spatial normalization has been applied – and several Volume Time Course (VTC) files that correspond to different combinations of preprocessing steps applied to a single localizer run.

 

Your task for this tutorial is to analyse these variably preprocessed functional scans to localise the fusiform face area (FFA). Using region of interest (ROI)-based GLM analysis, you will then compare beta weights and residuals resulting from a specific contrast.

 

The instructions below outline our approach using unprocessed data. You will first follow these instructions, then repeat them for each of the other preprocessed VTC files.

Specific Instructions

 

1.    Open the provided anatomical scan (VMR) file.

2.    From the Analysis menu, select Link Volume Time Course (VTC) File...

2.1. Click Browse, then select and open the VTC file (e.g. Loc1-S1R1_UNPROCESSED.vtc)

2.2. Click OK - you have now loaded 4-dimensional time course data that can be modelled using GLM analysis.

3.    From the Analysis menu, select General Linear Model: Single Study...

3.1. Click the Define Preds button.

3.2. Check the Show all checkbox under Predictors. You will now see the convolved predictors that will be used to model the data, as shown below.

3.3. Click GO to fit the model.

 

 

Once the model has been fit, you will see a default statistical map showing voxels in which a significant portion of the recorded signal was explainable in terms of the model.

 

4.    To localise the FFA, apply a contrast to specify an alternative hypothesis test that reveals significant activation for faces compared to other stimuli.

4.1. From the Analysis menu, select Overlay General Linear Model...

4.2. Next to Predictor Nr. 1, 3, and 4 in the Overlay GLM Contrasts window, click  once each to flip them to .

4.3. Click the Balance button to ensure that a balanced hypothesis test will be performed.

4.4. Click OK.

 

 

Notice that there are now far fewer voxels that qualify under this contrast. Why is that? What area of the brain could we suspect was activated by all stimulus categories?

 

Left: Statistical map using default contrast. Right: Statistical map using [3*Face > Hand + Body + Scrambled]

5.    To localise the FFA, we might need to loosen our statistical thresholds. Use the Increase / Decrease Thresholds buttons to change the limits imposed on the statistical maps.

5.1. Look for signs of positive t-statistics (orange to yellow) where you might expect to find the FFA. If you aren’t sure where the FFA might be located, try searching for it on Neurosynth.

6.    Once you are confident that you’ve located the FFA, apply a more conservative statistical threshold. For this tutorial, we will apply the same statistical threshold for all analyses. This way, a consistent comparison of betas and residuals can be made once we perform ROI analysis.

6.1. From the Analysis menu, select Overlay Volume Maps...

6.2. In the Volume Maps window, select the Statistics tab.

6.3. Under Thresholding, select Use statistic value

6.4. Move to the Map Options tab

6.5. Check the Bonferroni checkbox, and specify a p value of 0.05

6.6. Click Apply, then close the window.

 

 

 

The resulting statistical map is very conservative, but still shows activation where we might expect to find the FFA. Bonferroni correction - https://en.wikipedia.org/wiki/Bonferroni_correction - is a very conservative solution to the multiple comparisons problem, and can be used to define tightly-thresholded ROIs.

 

7.    Define a ROI for the FFA based on your analysis so far. If you are still having trouble finding the FFA, use 3D Volume Tools to navigate to system coordinates X=52, Y=173, Z=123.

7.1. Right-click over the centre of one of the crosshairs, and select Show ROI Time Course

7.2. In the ROI Signal Time Course window, click the green square in the bottom left to expand the window.

7.3. Click the ROI GLM button.

 

 

7.4. Select and open the design matrix file (SDM) that was automatically created when you fit the most recent GLM. Its name should end in autosave.sdm

7.5. Under Correction / output options, ensure that GLM graphs, Beta Plot, and GLM tables are checked. Ensure that Autocorrelation graphs and Serial correlation correction are unchecked.

7.6. Move to the Contrasts tab.

7.7. Toggle the predictor +/- boxes to specify a contrast that is positive for Face and negative for all other experimental predictors.

7.8. Click the Balance button.

7.9. Click the Fit GLM button.

 

 

The resulting model for this ROI can be analysed in several ways, and BrainVoyager provides summaries of multiples analyses. The following figures and tables should appear.

 

 

Blue: The average signal from the chosen ROI. Green: The explained signal in terms of the model. Red: The residuals, or error.

Instructions for Completed Steps

 

The following steps were used to create the preprocessed VTC files that are provided for this tutorial.

 

1.    Open the unprocessed FMR file, e.g. P01_Loc1-S1R1.fmr

1.1. From the Analysis menu, select FMR Data Preprocessing

1.2. Click Advanced >> to reveal the full dialog.

This dialog provides a variety of functional preprocessing options. For any combination of preprocessing options selected, new Slice Time Course (STC) and FMR files will be produced. Though STC files are very large, this process ensures that preprocessing does not overwrite your unprocessed files.

1.3. Under preprocessing options, check the options you wish to apply.

1.4. If you selected Temporal Filtering, for example, click Options... to adjust additional parameters. For this tutorial, default parameters were always used, except for one example in which an 8mm spatial smoothing filter was used, as opposed to a 4mm filter.

1.5. Click GO to apply your selected preprocessing steps.

 

2.    For this tutorial, we are only using data from one subject, and so no spatial normalization or alignment will be performed. See Spatial Normalization and Group GLM tutorial for these steps. Instead, we will align our preprocessed FMR and STC files to the same subject’s anatomical scan.

2.1. Open the anatomical scan (VMR) file, e.g. P01_Anat-S1_BRAIN_IIHC.vmr

2.2. If the 3D Volume Tools window is not open, click the blue cube button on the left-hand side of the main window to open it. Click the Full Dialog > button to reveal more options. Then select the Coregistration tab.

 

 

3.    Use the FMR-VMR coregistration controls to align your newly preprocessed functional data (FMR) to the same subject’s anatomical scan (VMR). This alignment step is important for comparing the results of various preprocessing step combinations or pipelines.

3.1. Click the Select FMR... button, then select and open the preprocessed FMR file you want to align.

3.2. Click the Align... button. This will open the FMR-VMR Coregistration window. By default, gradient-based, 9-parameter alignment will be performed. Note that this alignment process does not modify your FMR, STC, or VMR files. Instead, two additional files are produced that contain all information needed to reproduce the alignment. This is important because alignment processes are not always optimal, or even successful.

3.3. Click GO to perform the alignment and generate two transformation (TRF) files: one for initial alignment (IA), and one for fine-tuning alignment (FA).

 

4.    Use sanity checks to evaluate the quality of the alignment. In the lower row of images, you will see one volume of functional data (green) overlaying the anatomical scan (grey). Check that anatomical landmarks, such as the corpus callosum, large ventricles, and cortical sulci are reasonable aligned. For any study, it is important to always visually/manually assess the quality of alignments.

 

 

 

5.    Once you are satisfied with the quality of the alignment, create a new Volume Time Course (VTC) file that combines your preprocessing and alignment steps into a single 4-dimensional (X, Y, Z, time course) object.

5.1. From the Analysis menu, select Create Normalized VTC From FMR Data...

5.2. Select the preprocessed FMR file

5.3. Select the Initial Alignment TRF file – this file’s name should end in _IA.trf

5.4. Select the Fine-Tuning Alignment TRF file – this file’s name should end in _FA.trf

5.5. Click GO to create the VTC file. Note that you can choose a file name different than the default name – but always be careful to use consistent file names to avoid confusion.
